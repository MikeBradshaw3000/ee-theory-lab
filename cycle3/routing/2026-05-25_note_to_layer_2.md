## The corrected observable — where it stands

The corrected work is a **co-equal pair** of state-based, density-adjusted spatial observables, held co-equal on purpose:
- `Psi_meanI_state` — time-average over an earned window of per-tick Moran's I over active-state values.
- `Psi_persistence_I` — Moran's I over the persistence grid (fraction of window ticks each cell is active).

The v2 control battery (committed as Cycle 2's closing state) *produces* the designed discrimination on five synthetic cases — no substrate dependence: positive control (stable cluster) high on both; both negative controls (random low-ρ, random high-ρ) near zero; the uniform-high-persistence degeneracy correctly near zero; and the transient-wave case showing the pair *diverge by design* — `Psi_meanI_state` high (~0.71), `Psi_persistence_I` zero. The permutation-null z-score is the separator that distinguishes true signal from the uniformity degeneracy.

This is an **L1/L2 result only**: the apparatus computes the intended quantities and discriminates correctly on synthetic controls. It says nothing about what any real-substrate value *means*. Interpretation on the real substrate stays withheld.

## The open question Cycle 3 inherits — yours to engage, not mine to settle

Which observable, if either, *is* Regime-II coherence — per-tick state organization that persists, or the same cells persistently clustered — turns on coherence-as-relational-mutual-reinforcement (the baked-in hinge), **not** on which statistic is more robust. This is held deliberately open. It must not be settled by statistic robustness, by which observable is cleaner to compute, or by script naming.

The transient-wave case is where the pair genuinely diverges. That divergence is **data**, not a defect to smooth into one reading. If anything, it is the most informative case for the ontology.

## Carry-forward cautions

### New from the 2026-05-28 session

- **BOM check must be paired with size check.** A zero-byte file passes the BOM-False check trivially (no bytes match the BOM sequence). When verifying a downloaded file before placement, check BOM AND file size. Caught this session via a Pane 3 misfire that returned "Line count: 1" and "Last non-empty line: [empty]" — diagnostic Pane 4 with `Resolve-Path` and byte-size check resolved it (the file was actually 15,851 bytes at destination; Pane 3's `[System.IO.File]::ReadAllText` had been operating on a different cwd via relative path).

- **`.NET` I/O calls need `Resolve-Path` first.** `.NET` resolves relative paths against process cwd, not PS location. `Test-Path "cycle3\foo"` correctly uses PS location; `[System.IO.File]::ReadAllText("cycle3\foo")` does not. Always `Resolve-Path` to get the absolute path before any `.NET` call. Kit Section 7 step 2 names this for `Get-ChildItem` vs `git status` reading; it generalizes to all `.NET` I/O.

- **"Top-level venv" is ambiguous between repo root and workspace root.** The 2026-05-27 anchor said "top-level venv" without disambiguation; this session's run failed pre-flight because the venv was being looked for at the repo root. The canonical venv is at `C:\Users\vkz244\EE_Theory_Lab\venv` — workspace top-level, parent of the repo. There is also a `venv_cycle1_archive` next to it (preserved Cycle 1 venv; do NOT activate). Use the call operator `&` with absolute path to invoke `Activate.ps1`, then verify via `$env:VIRTUAL_ENV` + Python-prefix comparison before trusting activation.

- **Substrate-run sufficiency-test must aggregate flag distributions per (rule, lambda_id), not just spot-check numerical means.** Layer 1 missed Layer 3's overstated [1,2] claim (Layer 3 said earned-steady on 5/5 seeds; CSV shows 1/25 windows) because the sufficiency-test verified mean values but not the flag-pattern claim. Carry-forward: in future sufficiency-tests on substrate-run outputs, explicitly aggregate boolean flag counts per band/per seed. Confirming `mean_rho ≈ X` is not the same as confirming `Steady_State_Candidate True on 5/5`.

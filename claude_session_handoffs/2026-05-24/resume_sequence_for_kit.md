# Layer 1 Resume Sequence

*Paste into the kit / README / resume instructions wherever the Layer 1 startup steps
live. This establishes the load order: purpose before machinery.*

On instantiation, Layer 1 proceeds in this order:

1. **HEAD / handoff verification.** Confirm instance/mode state and the integrity of
   the foundational set and canonical record.
2. **GOAL_ANCHOR.md** — load *before* STANDING_ITEMS. Establishes the purpose (the
   Regime-II manuscript and the ABM's role in defeating "baked in") before the machinery.
3. **STANDING_ITEMS** — load immediately after, now read in light of the purpose.
4. **Run the Goal Anchor operating rule before touching items:**
   1. What is the nearest live move bearing on Regime-II-as-structural?
   2. Which layer should carry the next piece of work?
   3. What can be safely delegated to Layer 3 now without weakening the protocol?
   4. What must remain with Mike / Layer 1 / Layer 2?

Rationale: the continuity machinery is far more explicit than the manuscript goal. A
fresh instance that meets the items before the purpose continues custodial work by
default. Purpose first, then items, keeps the protocol visibly in service of the
manuscript claim — and protects Layer 1's context budget by routing work to the layer
that should carry it.
